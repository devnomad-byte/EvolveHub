# EvolveHub 后端服务部署文档

## 架构概览

| 服务 | 容器名 | 端口 | 路径前缀 | 说明 |
|------|--------|------|----------|------|
| Gateway | evolvehub-gateway | 8080 | / | 统一网关，路由分发 |
| Auth | evolvehub-auth | 8081 | /api/auth | 认证授权服务 |
| AI | evolvehub-ai | 8082 | /api/ai | AI 平台服务 |
| Admin | evolvehub-admin | 8083 | /api/admin | 后台管理服务 |

**前提条件**：服务器已独立部署 PostgreSQL、Redis、Minio 等基础设施。

---

## 一、服务器环境准备

### 1.1 安装 Docker

```bash
# 如果尚未安装
curl -fsSL https://get.docker.com | sh
systemctl enable docker && systemctl start docker
```

### 1.2 创建部署目录

```bash
mkdir -p /opt/evolvehub && cd /opt/evolvehub
```

### 1.3 登录阿里云镜像仓库

```bash
docker login xxx.cr.aliyuncs.com
# 输入阿里云容器镜像服务的用户名和密码
```

---

## 二、配置文件

需要在 `/opt/evolvehub` 目录下放置两个文件：

### 2.1 环境变量文件 `.env`

```bash
cat > .env << 'EOF'
# ==================== 镜像仓库 ====================
DOCKER_REGISTRY=xxx.cr.aliyuncs.com/s_m
IMAGE_TAG=latest

# ==================== PostgreSQL ====================
DB_HOST=host.docker.internal
DB_PORT=5432
DB_USERNAME=admin
DB_PASSWORD=你的数据库密码

# ==================== Redis ====================
REDIS_HOST=host.docker.internal
REDIS_PORT=6379
REDIS_PASSWORD=

# ==================== Minio / S3 ====================
S3_ENDPOINT=http://host.docker.internal:9000
S3_ACCESS_KEY=minioadmin
S3_SECRET_KEY=minioadmin
EOF
```

> **说明**：`host.docker.internal` 让容器访问宿主机上的基础设施。  
> 如果 Docker 版本不支持，替换为服务器内网 IP（如 `172.17.0.1` 或实际 IP）。

### 2.2 服务编排文件 `docker-compose-services.yml`

将项目中的 `backend/docker-compose-services.yml` 文件上传到此目录即可。

---

## 三、部署操作

### 3.1 首次部署

```bash
cd /opt/evolvehub

# 拉取所有镜像
docker compose -f docker-compose-services.yml pull

# 启动所有服务
docker compose -f docker-compose-services.yml up -d
```

### 3.2 更新部署（有新版镜像时）

```bash
cd /opt/evolvehub

# 拉取最新镜像
docker compose -f docker-compose-services.yml pull

# 重启服务（只会重建有变化的容器）
docker compose -f docker-compose-services.yml up -d
```

### 3.3 部署指定服务

```bash
# 只更新 auth 和 admin
docker compose -f docker-compose-services.yml pull evolvehub-auth evolvehub-admin
docker compose -f docker-compose-services.yml up -d evolvehub-auth evolvehub-admin
```

## 四、运维命令

### 4.1 查看服务状态

```bash
docker compose -f docker-compose-services.yml ps
```

### 4.2 查看日志

```bash
# 查看所有服务日志
docker compose -f docker-compose-services.yml logs

# 实时跟踪指定服务日志
docker compose -f docker-compose-services.yml logs -f evolvehub-auth

# 查看最近 100 行
docker compose -f docker-compose-services.yml logs --tail 100 evolvehub-gateway
```

### 4.3 重启服务

```bash
# 重启单个服务
docker compose -f docker-compose-services.yml restart evolvehub-auth

# 重启所有服务
docker compose -f docker-compose-services.yml restart
```

### 4.4 停止服务

```bash
# 停止所有服务
docker compose -f docker-compose-services.yml down

# 停止单个服务
docker compose -f docker-compose-services.yml stop evolvehub-ai
```

## 五、开发端推送流程

在开发机（macOS）上执行：

```bash
cd backend/

# 构建全部服务并推送到阿里云
./deploy.sh push

# 只推送指定服务
./deploy.sh push auth gateway
```

推送完成后，在服务器执行 `pull + up -d` 即可完成更新。

## 七、故障排查

| 问题 | 排查方式 |
|------|----------|
| 容器启动后立即退出 | `docker logs evolvehub-auth` 查看启动日志 |
| 连接数据库失败 | 检查 `.env` 中 `DB_HOST` 是否正确，确认 PostgreSQL 允许远程连接 |
| 连接 Redis 失败 | 检查 Redis 是否绑定了 `0.0.0.0`，防火墙是否放行 6379 |
| 服务间调用 404 | 确认 Gateway 路由配置，所有服务需在同一 `evolvehub-net` 网络 |
| 拉取镜像失败 | 重新执行 `docker login` 登录阿里云仓库 |
